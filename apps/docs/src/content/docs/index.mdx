## A memory layer that earns its claims

Continuum is an open-source [Model Context Protocol](https://modelcontextprotocol.io)
engine that gives AI coding assistants a persistent, verifiable memory of a
project: its state, its decisions, its history, across every session.

Its moat is a discipline, not a feature: **work is not "done" until a shell
command exits 0 to prove it.**

<CardGrid stagger>
  <Card title="Verify-then-dissolve" icon="approve-check">
    Every checkpoint carries a `verifyCommand`. State that can't be re-proven
    by a green shell exit is never marked complete; it stays a witness until then.
  </Card>
  <Card title="Progressive disclosure" icon="seti:db">
    A 3-layer retrieval surface (search → timeline → full-fetch) cuts retrieval
    tokens by a measured **~2.85×** (up to 5.3× when an answer needs a single
    record), benchmarked on this repo's own history. Reproduce it:
    `node scripts/benchmark-token-savings.mjs`.
  </Card>
  <Card title="Local-first, swappable" icon="laptop">
    SQLite + FTS5 today, RuVector vector search next, behind one
    `StorageBackend` seam. Your history stays on your machine by default.
  </Card>
  <Card title="Privacy at the write" icon="seti:lock">
    A secret-scrubbing filter sits on the single write choke-point, redacting
    known credential patterns before anything is persisted.
  </Card>
</CardGrid>
